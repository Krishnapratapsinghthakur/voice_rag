Here is a step-by-step guide to creating a simple REST API using Django and Django REST Framework (DRF).

We will build a simple "Student" API where you can Create, Read, Update, and Delete (CRUD) student records.

Step 1: Set Up the Virtual Environment
First, create a folder for your project and set up a virtual environment to keep your dependencies isolated.

Open your terminal (Command Prompt or PowerShell on Windows, Terminal on Linux/Mac).

Run the following commands:

Bash

# Create a new directory
mkdir django_api_demo
cd django_api_demo

# Create the virtual environment
python -m venv venv

# Activate the virtual environment
# Windows:
venv\Scripts\activate
# Linux / macOS:
source venv/bin/activate
(You will see (venv) appear at the start of your command line, indicating it is active.)

Step 2: Install Django and REST Framework
Install Django and the toolkit that makes building APIs easy.

Bash

pip install django djangorestframework
Step 3: Start the Project
Create the Django project and a separate "app" for your API logic.

Bash

# Create the project (the "." at the end installs it in the current folder)
django-admin startproject myproject .

# Create an app named 'api'
python manage.py startapp api
Step 4: Configure Settings
You need to tell Django that you are using the REST Framework and your new api app.

Open myproject/settings.py.

Find the INSTALLED_APPS list and add 'rest_framework' and 'api':

Python

# myproject/settings.py

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    
    # Add these two lines:
    'rest_framework',
    'api', 
]
Step 5: Create the Database Model
Let's define what data we are storing. We will create a simple Student model.

Open api/models.py.

Add this code:

Python

from django.db import models

class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
    email = models.EmailField()

    def __str__(self):
        return self.name
Apply these changes to the database: Go back to your terminal and run:

Bash

python manage.py makemigrations
python manage.py migrate
Step 6: Create the Serializer
Serializers convert your database models into JSON (which APIs use).

Create a new file inside the api folder called serializers.py.

Add this code:

Python

# api/serializers.py
from rest_framework import serializers
from .models import Student

class StudentSerializer(serializers.ModelSerializer):
    class Meta:
        model = Student
        fields = '__all__'  # This includes all fields (id, name, age, email)
Step 7: Create the View
The "View" handles the logic. We will use a ModelViewSet, which automatically writes the code for Create, Read, Update, and Delete.

Open api/views.py.

Add this code:

Python

# api/views.py
from rest_framework import viewsets
from .models import Student
from .serializers import StudentSerializer

class StudentViewSet(viewsets.ModelViewSet):
    queryset = Student.objects.all()
    serializer_class = StudentSerializer
Step 8: Set Up URLs
Finally, we need to wire up the URL so you can access the API in your browser.

Open myproject/urls.py (the main project URL file).

Update it to look like this:

Python

# myproject/urls.py
from django.contrib import admin
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from api.views import StudentViewSet

# Create a router and register our viewset with it.
router = DefaultRouter()
router.register(r'students', StudentViewSet)

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api/', include(router.urls)), # This includes all CRUD URLs automatically
]
Step 9: Run the Server and Test
You are done! Now let's test it.

Run the server:

Bash

python manage.py runserver
Open your web browser and go to: http://127.0.0.1:8000/api/students/

What you can do now:

List (GET): You will see an empty list [].

Create (POST): Scroll down to the form at the bottom, enter a Name, Age, and Email, and click "POST".

Update/Delete: After creating a student, click on the URL for that specific student (e.g., .../students/1/) to see buttons for "DELETE" or "PUT" (update).

1. Create the Component (SignIn.jsx)
This component handles the user input.

JavaScript

import { useState } from 'react';

function SignIn() {
  const [formData, setFormData] = useState({
    email: '',
    password: ''
  });

  const handleChange = (e) => {
    setFormData({
      ...formData,
      [e.target.name]: e.target.value
    });
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('Login submitted:', formData);
    // Add your login logic here (e.g., API call)
  };

  return (
    <div>
      <h1>Sign In Dashboard</h1>
      <form onSubmit={handleSubmit}>
        <div>
          <label htmlFor="email">Email: </label>
          <input
            type="email"
            id="email"
            name="email"
            value={formData.email}
            onChange={handleChange}
            required
          />
        </div>
        <br />
        <div>
          <label htmlFor="password">Password: </label>
          <input
            type="password"
            id="password"
            name="password"
            value={formData.password}
            onChange={handleChange}
            required
          />
        </div>
        <br />
        <button type="submit">Sign In</button>
      </form>
    </div>
  );
}

export default SignIn;
2. Set up the Route (App.jsx)
This connects the component to the /signin URL.

JavaScript

import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';
import SignIn from './SignIn';

function App() {
  return (
    <BrowserRouter>
      {/* Navigation link just to help you test it easily */}
      <nav>
        <Link to="/signin">Go to Sign In</Link>
      </nav>
      <hr />

      <Routes>
        {/* This renders the SignIn component when the URL is /signin */}
        <Route path="/signin" element={<SignIn />} />
        
        {/* Optional: A default home route so the screen isn't empty on load */}
        <Route path="/" element={<h2>Home Page</h2>} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
